📌 2026 年度工作计划

保障业务基本盘 · 电商与创作者工具双线 Agent 场景验证 · 沉淀 AI 时代团队能力底座

📋
一、总体判断

2026 年，团队工作需要同时守住业务基本盘，并通过简单AI验证新的 AI 产品机会。整体方向不再发散为多个 AI 场景探索，而是聚焦三件事：

  1. 业务基本盘：继续保障搜狐视频推荐和搜狐媒体业务。视频推荐围绕播放规模、时长、人均等核心指标优化；媒体业务在流量增长空间有限的情况下，重点提升推荐效果、产品迭代效率和 AIGC 有效流量经营。
  2. 产品方向：简单AI收敛为两条 Agent 产品线：
    1. 电商 Agent：验证商品卖点、营销文案、商品图/素材生成、平台适配等高频任务的提效价值。
    2. 创作者工具 Agent：验证选题、标题、封面、正文、分发适配等连续创作任务的支持能力。
  3. 组织方式：AI 创新产品采用内部闭环推进，每条线配置 1 个产品 Owner + 1 个技术 Owner。产品 Owner 负责场景、MVP、指标和复盘；技术 Owner 负责 Agent 流程、tools 调用、模型调用和工程串联；前端/后端仅按需提供已有系统能力和线上稳定性支持。

O2：支撑媒体业务指标优化和产品迭代，提升 AIGC 内容增长能力媒体业务基本盘推荐算法、运营共同负责媒体推荐支撑、产品迭代、AIGC 有效流量经营合并管理
O3：围绕电商和创作者工具两条线，跑通简单AI Agent 产品闭环新产品探索 / 组织机制升级产品运营、智能中台共同负责，推荐算法按需参与同时承接方向判断、场景验证、数据复盘和机制沉淀

三、O1：保障搜狐视频推荐核心指标
目标说明

O1 是搜狐视频推荐业务基本盘目标。重点关注播放规模、播放总时长、人均播放时长指标。

KR1：搜狐视频推荐核心指标达到目标值

  1. 播放数：播放规模达到96万 ，短视频播放规模达到90万
  2. 播放时长：播放总时长达到70万分钟，短视频总时长达到50万
  3. 人均播放时长：短视频人均播放时长达到 9.5分钟

KR2：视频推荐重点场景取得可验证优化结果

  1. 视频推荐重点场景完成 60 个实验推全或策略优化
  2. 核心指标提升15%
📰
四、O2：支撑搜狐媒体业务基本盘，提升推荐效果与 AIGC 有效流量规模
目标说明

O2 是搜狐媒体业务基本盘目标。推荐算法仍需要支撑媒体核心场景指标优化，并配合媒体业务产品迭代和稳定运行。

AIGC 内容增长仍服务于媒体业务基本盘，但核心矛盾不是产能不足，而是高流量方向风险高、低风险垂类流量弱、已有工作流缺少持续优化。后续应从增加工作流转向垂类运营、效果复盘。

KR1：媒体核心推荐场景取得可验证优化结果

  1. 媒体核心推荐场景完成 15 个实验推全或指标优化
  2. 核心消费（页面pv转化率或人均阅读时长）指标提升 10 %

KR2：媒体业务产品迭代按节奏交付

  1. 媒体重点产品需求双周交付率达到 90%

KR3：AIGC 内容增长带来有效流量并提升运营效率

  1. AIGC 内容带来的新增有效 PV 达到 日均200万，投诉率控制在万分1以下
  2. 千次 PV 成本降低 20%
🤖
五、O3：围绕电商和创作者工具两条线，跑通简单AI Agent 产品闭环
目标说明

O3 是今年最核心的新产品探索目标，同时承担三个作用：

  1. 将简单AI下阶段产品方向收敛为电商方向和创作者工具方向；
  2. 建立团队内部从 Agent 能力开发、场景验证到数据复盘的完整闭环；
  3. 缩短新场景从想法到场景验证、反馈复盘和能力沉淀的周期。

KR1：完成至少一个电商营销 Agent 场景验证

  1. 验证商品卖点、营销文案、商品图/素材生成、平台适配等任务的内容生产效率提升。

KR2：完成至少一个创作者工具方向 Agent 场景验证

  1. 验证选题、标题、封面、正文、分发适配等任务的连续创作流程支持。
共同验收口径
  1. 每条线完成一个明确 MVP 任务的 Agent 版本，并进入种子用户或内部业务场景验证；
  2. 每条线基于不少于 20 个真实业务样例完成验证，沉淀有效样例、失败 case 和典型反馈；
  3. 每条线形成继续投入、调整方向或暂停的复盘结论。

KR3：形成 AI 创新产品内部闭环机制

  1. 每条线都有明确的产品 Owner 和技术 Owner。
  2. 产品 Owner 负责场景定义、MVP 边界、核心价值行为、验收标准和复盘。
  3. 技术 Owner / Agent 开发工程师负责 Agent 流程、上下文管理、tools 封装/调用、模型调用和工程串联。
  4. 运营负责样例、业务规则、风险红线、结果评估和失败 case 归档。
  5. 前端/后端不作为新场景主交付方，仅按需支持已有系统能力、接口和线上稳定性。
🏗️
六、BTH O4：沉淀 AI 时代团队能力底座
目标说明

O4 是愿景型目标，团队后续能否持续提供业务帮助，不只取决于单个项目是否成功，也取决于是否能沉淀可复用的底座能力和知识资产。能力底座也要包括 Agent 开发、tools 封装以及场景验证的能力。

成功信号
  1. 沉淀可复用的 AI skills、工作流、prompt、模板或 SOP；
  2. 沉淀AI 项目知识库，覆盖选题、调研、demo、场景验证、数据复盘、风险控制等环节；
  3. 沉淀 Agent 场景定义模板、用户任务拆解模板、领域指令模板、场景验证 checklist、效果评估口径、风险红线和失败 case 库；
  4. 智能中台沉淀 Agent 规划、反问、能力调用、结果生成、反馈优化、成本观测和性能优化等能力组件；
  5. 团队成员在新项目中不再从零开始，能够基于已有能力快速组合和迭代。
👥
七、各团队下阶段重点工作方向

1. 产品团队

产品团队的重点从"提功能需求、体验优化"转向"围绕电商和创作者工具两条线判断哪些用户任务值得被 Agent 化"，同时需要配合视频和媒体业务保留必要的产品迭代支持。

下阶段重点工作：

  1. 明确简单AI两条产品线的用户人群和任务边界：电商方向先围绕商品卖点、营销文案、商品图 / 素材生成、平台适配；创作者工具方向围绕选题、标题、封面、正文和分发适配；
  2. 将用户任务拆清楚，明确每个任务的输入、输出、核心价值行为和失败边界；
  3. 定义每个场景的 MVP 边界、核心价值行为、场景验证指标和停止条件；
  4. 配合视频推荐和媒体业务，明确必须支持的重点需求和交付节奏；

2. 运营团队

运营团队的重点从"模板运营"转向"让 Agent 懂业务，并持续迭代效果"，同时继续承担 AIGC 内容增长的有效流量经营。

下阶段重点工作：

  1. 提供正向样例、反向样例和失败 case。
  2. 梳理业务规则、平台规范、风险红线和禁用表达。
  3. 参与生成结果评估，判断结果是否可用。
  4. 跟进种子用户反馈，沉淀真实使用问题。

3. 智能中台团队

智能中台团队的重点在"大模型能力供给"的基础上，投入更多精力到"Agent 能力层建设"。

下阶段重点工作：

  1. 负责 Agent 上下文管理、流程编排、反问逻辑和多轮调整。
  2. 负责 tools 的封装、调用和串联。
  3. 负责模型能力接入、效果调优和稳定性处理。
  4. 负责将 demo 推进到可演示、可验证、可复盘的交付状态。

4. 推荐算法团队

推荐算法团队的重点分为两层：一是独立保障视频推荐长期核心指标，二是支撑媒体业务核心场景优化，并按需参与 AI 新场景建设。

下阶段重点工作：

  1. 保障搜狐视频推荐核心指标，包括播放规模、播放总时长、人均播放时长等；
  2. 推动视频推荐重点场景实验和策略优化；
  3. 支撑媒体核心推荐场景的算法优化和产品迭代；
  4. 控制基本盘投入范围，避免低增量事项占用过多资源；
  5. 按需参与 AI 新场景建设，重点提供推荐、分发、数据分析和生成式推荐经验。
